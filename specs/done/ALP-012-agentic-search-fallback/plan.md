# Implementation Plan: Agentic Search with Web Fallback

**Branch**: `feature/ALP-012-agentic-search-fallback` | **Date**: 2026-03-05 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/ALP-012-agentic-search-fallback/spec.md`

## Summary

Replace the current confirmation-based fallback flow in the RAG chat with a single LangChain ReAct agent that autonomously searches the user's knowledge base first, falls back to Serper web search when KB results are insufficient, and includes source labels in non-KB responses. The chat UI gets a "Use websearch" toggle. Rate limiting controls web search costs. A confidence-based fast path optimization skips the agent loop for high-confidence KB hits.

## Technical Context

**Language/Version**: Python 3.12+ (backend), TypeScript (Next.js 15 frontend)
**Primary Dependencies**: LangChain 0.3+, LangGraph (prebuilt agents), langchain-community (GoogleSerperAPIWrapper), ChatOpenAI, sse-starlette
**Storage**: No new database tables. In-memory rate limiter. Env var configuration.
**Testing**: Manual verification via quickstart.md scenarios
**Target Platform**: Linux server (backend), browser (frontend)
**Project Type**: Web application (Python backend + Next.js frontend)
**Performance Goals**: KB-only responses ≤5s, web search responses ≤10s
**Constraints**: Web search costs bounded by configurable per-user rate limit
**Scale/Scope**: Single-user concurrent, same as current chat

## Key Architecture Decisions

### Agent Design

- **Single ReAct agent** via `create_react_agent` from `langgraph.prebuilt`
- **Two tools**: `search_knowledge_base` (wraps existing VectorStoreService) and `web_search` (Serper)
- **Per-request agent creation**: Tools are created per-request with the user's vectorstore instance injected via closure. The LLM instance can be shared.
- **System prompt**: Instructs agent to always try KB first, only use web search if KB insufficient, and label non-KB responses with source prefix.

### Streaming

- LangGraph `stream_mode="messages"` with metadata filtering to skip tool call messages
- Fallback to `astream_events(version="v2")` if streaming regression is encountered
- SSE transport unchanged (`sse-starlette`)
- Frontend SSE parsing unchanged (already manual `data:` line parsing)

### Confidence-Based Fast Path (US5/P3)

- Before creating the agent, perform a quick similarity search
- If top result score ≥ `RAG_CONFIDENCE_THRESHOLD` (default 0.75), use the current non-agentic pipeline (direct LLM call with context)
- If below threshold, create and invoke the full agent
- This avoids 1-2 extra LLM calls for the majority of queries

### Web Search Availability

- Backend checks if `serper_api_key` is set in config
- New `GET /v1/api/chat/config` returns `{"web_search_available": bool}`
- Frontend fetches on chat page load, disables toggle if unavailable
